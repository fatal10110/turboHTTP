# Phase 22b.3: Review Fixes

**Date:** 2026-03-21  
**Phase:** 22b.3 response trailers  
**Status:** Review findings fixed and revalidated

## What was implemented

Applied the follow-up fixes from `Development/docs/phases/phase22b/phase-22b.3-review.md`:

1. Tightened trailer API naming and parser/body-source parity.
   - renamed `TryAddResponseTrailer(...)` to `AddResponseTrailer(...)`
   - aligned the chunked trailer reader naming so both code paths now use `ReadChunkedTrailersAsync(...)`

2. Added aggregate trailer-size enforcement to both HTTP/1.1 trailer readers.
   - introduced `Http11ResponseParser.MaxTotalTrailerBytes` with a 32KB cap
   - enforced the same cap in the buffered parser path and the streaming `Http11ResponseBodySource` path

3. Expanded prohibited response-trailer filtering.
   - added `Content-Location` to the shared denied response-trailer set in `TrailerFieldValidator`

4. Filled the missing review coverage.
   - added parser coverage for fragmented trailer reads
   - added parser/body-source coverage for aggregate trailer-size overflow
   - added streaming coverage for CR injection rejection
   - added the abort-path trailer task fault test
   - added the buffered HTTP proxy trailer preservation test

5. Fixed a real buffered chunked-response regression exposed by the new proxy test.
   - `RawSocketTransport.EmitParsedResponseAsync(...)` returned segmented parsed responses to the pool before passing `StatusCode` and `Headers` to the handler
   - chunked buffered responses therefore surfaced a default `0` status and lost headers on that path
   - the transport now snapshots status/headers before returning the pooled parsed response

## Decisions and trade-offs

1. Kept aggregate trailer-size enforcement shared by constant rather than duplicating magic numbers.
   - this keeps buffered and streaming behavior aligned

2. Preserved the existing per-line limit in addition to the new total-size limit.
   - oversized individual lines still fail fast before aggregate accounting

3. Fixed the proxy buffered regression in the transport instead of weakening the new test.
   - the failure was a legitimate pooled-object lifetime bug on segmented buffered responses
