# Phase 22a.1: Handler Contract Rewrite, API Clean Break, and AOT/Test Primitives

**Date:** 2026-03-17  
**Phase:** 22a.1 final implementation slice  
**Status:** Implemented in code; external prerequisite/device validation still pending

## What Was Implemented

Completed the remaining 22a.1 code-side work after the earlier request-content and response-foundation slices:

1. Rewrote the runtime handler contract to:
   - `OnRequestStart(...)`
   - `OnResponseStartAsync(int, HttpHeaders, IResponseBodySource, RequestContext)`
   - `OnResponseError(...)`
2. Added buffered and streaming dispatch bridges:
   - `BufferedDispatchBridge`
   - `StreamingDispatchBridge`
3. Replaced the old collector flow with `BufferedResponseCollectorHandler` over `IResponseBodySource`.
4. Added `BufferedResponseBodySource` as the buffered body-source implementation used by the current transport/testing compatibility path.
5. Migrated shipping runtime handlers, testing transports, cache replay/store, and the HTTP/1.1 + HTTP/2 transport emit paths to the new handler contract.
6. Split the public client API to:
   - `UHttpClient.SendBufferedAsync(...)`
   - `UHttpClient.SendStreamingAsync(...)`
   - `UHttpRequest.SendBufferedAsync(...)`
   - `UHttpRequest.SendStreamingAsync(...)`
7. Removed the temporary HTTP compatibility API from the public request/client surface:
   - removed `UHttpClient.SendAsync(...)`
   - removed `UHttpRequest.SendAsync(...)`
   - removed `UHttpRequest.Body`
8. Migrated remaining shipping runtime HTTP callsites and request-body consumers to the `Content` model and explicit buffered send APIs.
9. Added `MockResponseBodySource` for 22a.1 testing and rewired the streaming-response tests to validate opening a response over a non-buffered body source.
10. Added `SingleReaderChannel<T>` in `TurboHTTP.Transport` as the 22a.1 HTTP/2 queue primitive.
11. Added 22a.1 linker metadata in `Runtime/Core/link.xml` and updated `Runtime/Transport/link.xml`.

## Decisions / Trade-Offs

1. **22a.1 is now a real clean break at the HTTP public API level.** The repo no longer carries `UHttpRequest.Body` or HTTP `SendAsync(...)` as compatibility shims.
2. **Buffered compatibility remains transport-limited by design.** HTTP/1.1 and HTTP/2 send paths now fail clearly for non-buffered request bodies rather than pretending streaming upload support exists before 22a.2 / 22a.3.
3. **Test compatibility was kept test-only.** `LegacyHttpHandlerCompatExtensions` is wrapped in `UNITY_INCLUDE_TESTS` so old handler-style tests can be migrated incrementally without polluting shipping runtime assemblies.
4. **`MockResponseBodySource` intentionally supports both buffered and non-buffered test modes.** Single-buffer construction exposes `TryGetBufferedData`, while chunked construction validates streaming-response behavior without forced flattening.
5. **`SingleReaderChannel<T>` is added now but not integrated yet.** This satisfies the 22a.1 transport-boundary requirement without prematurely rewriting the current buffered HTTP/2 receive path before 22a.3.

### Findings Addressed In This Pass

| Rubric | Finding | Resolution |
|--------|---------|------------|
| Infrastructure | Public HTTP API still exposed ambiguous `SendAsync` and raw request-body access | Removed `SendAsync` and `UHttpRequest.Body`; migrated runtime/test callsites to explicit APIs and `Content` |
| Infrastructure | Handler contract rewrite must not leak callback failures or orphan response resources | `HandlerCallbackSafetyWrapper` now aborts body sources on callback failure and the buffered/streaming bridges own completion/failure paths |
| Infrastructure | AOT-facing 22a.1 abstractions need linker preservation | Added `Runtime/Core/link.xml` and updated `Runtime/Transport/link.xml` |
| Network | Runtime response delivery must move to a body-source contract before streaming transport phases | Transports/testing/cache now produce or wrap `IResponseBodySource` at the handler boundary |
| Network | Streaming response must be openable without full buffering | Added `MockResponseBodySource` and updated streaming-response tests to exercise a multi-chunk non-buffered source |
| Network | HTTP/2 queue primitive must stay transport-local | Added `SingleReaderChannel<T>` under `Runtime/Transport/Http2/` only |
