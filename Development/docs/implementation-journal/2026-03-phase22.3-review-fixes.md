# Phase 22.3: Review Fixes

**Date:** 2026-03-11
**Phase:** 22.3 follow-up (implementation review fixes)  
**Status:** Pass 1, Pass 2, and the actionable Pass 4 review findings are addressed in code; buffered decompression remains a documented deferred limitation, Unity/device validation still pending

## What Was Fixed

This pass closes the actionable findings from the updated Phase 22.3 review set:

1. Restored the missing `System.Threading` imports in `AdaptiveInterceptor.cs` and `BackgroundNetworkingPolicy.cs` so the Core assembly compiles again under Unity 2021.3 / C# 8.
2. Corrected retry terminal-attempt telemetry, and stopped retry loops from continuing after a retryable transport error was already delivered on a committed response path.
3. Hardened decompression with broader exception handling, a decompressed-size limit, multi-value `Content-Encoding` support, and `x-gzip` alias handling while preserving unsupported encodings untouched.
4. Removed the redirect completion-bridge hang when a spawned redirect dispatch completes successfully without ever delivering `OnResponseEnd(...)` or `OnResponseError(...)`.
5. Reduced hot-path allocations in observability by making logging preview allocation lazy, removing monitor `_responseBody?.ToArray()` capture, and caching `ReadOnlySequenceStream.Position`.
6. Bound cache background store/revalidation work to interceptor disposal via cancellation, and explicitly observed background tasks so faults cannot escape as unobserved task exceptions.
7. Fixed the remaining clone-ownership leaks on synchronous `next(...)` throws, wired `GracePeriodBeforeQueue` into background networking, and removed the `Cookie` merge `Split(';')` array allocation.
8. Added targeted regressions for decompression truncation/limits/encoding chains, retry committed-error behavior, redirect completion bridging, metrics 3xx classification, cache disposal cancellation, and background scope budget behavior.

## Pass 4 Follow-Up (2026-03-11)

This follow-up closes the actionable findings from the refreshed Pass 4 review:

1. Restored `RequestContext.Request` in `CacheInterceptor.RevalidateAsync` on both success and exception paths so outer teardown/observability never remains pinned to the internal conditional revalidation request.
2. Added a broad terminal-error fallback in `DecompressionHandler.OnResponseEnd` so unexpected buffered decompression failures, including downstream `OnResponseData(...)` callback exceptions, still end via `_inner.OnResponseError(...)`.
3. Hardened `RedirectHandler` completion bridging so synchronous downstream `OnResponseEnd(...)` / `OnResponseError(...)` failures fault the redirect completion source instead of degrading into the generic completion-bridge error.
4. Guarded `CacheStoringHandler` ownership transfer so detached response buffers are disposed if synchronous queueing fails before background mutation work takes over.
5. Decoupled monitor capture-failure throttling from `HistoryLock` and capped `MonitorHandler` response buffering to a bounded prefix sized for snapshot policies plus binary detection.
6. Added focused regressions for failed cache revalidation context restoration, decompression callback failure terminal delivery, and redirect downstream terminal-failure propagation.
