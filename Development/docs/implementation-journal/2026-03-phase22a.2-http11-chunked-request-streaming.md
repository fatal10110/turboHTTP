# Phase 22a.2 Step 2: HTTP/1.1 Chunked Request Streaming

**Date:** 2026-03-18  
**Phase:** 22a.2 Step 2  
**Status:** Implemented and validated in a runtime harness; Unity/package validation still pending

## What Was Implemented

Built the next HTTP/1.1 request-side streaming slice on top of the earlier known-length upload work:

1. `Http11RequestSerializer` now supports **unknown-length** request bodies by emitting `Transfer-Encoding: chunked`.
2. Unknown-length bodies stream through `RequestBodyReadSession` and are chunk-framed on the fly:
   - each chunk writes `<size-hex>\r\n`
   - then chunk data
   - then `\r\n`
   - final sequence is always `0\r\n\r\n`
3. Framing normalization now covers all Step 1 and Step 2 request-body cases:
   - empty bodies strip both framing headers
   - known-length bodies emit `Content-Length`
   - unknown-length bodies emit `Transfer-Encoding: chunked`
4. Added serializer tests and raw transport coverage for chunked uploads, including stripping a user-provided `Content-Length` when the body length is unknown.

## Decisions / Trade-Offs

1. **Chunked writes flush after each chunk for now.** This matches the 22a plan's default latency-first behavior; tuning knobs remain future work.
2. **Chunk-size formatting stays allocation-aware.** `Utf8Formatter` writes into a pooled header buffer so chunk framing does not allocate per chunk.
3. **Retry semantics stay at the Step 1 conservative gate.** Unknown-length non-replayable uploads are already protected by the replayability-based stale-retry guard; committed-byte-aware retry logic remains a later 22a.2 step.

### Network Pass

- HTTP/1.1 unknown-length uploads now match RFC-style chunk framing for the implemented scope
- Terminal chunk sequence is emitted exactly once
- Request framing precedence is now explicit on the serializer side:
  - transport decides framing
  - user-supplied framing headers are normalized away
- Remaining request-side protocol gaps are still deferred:
  - `Expect: 100-continue`
  - committed-byte-aware retry decisions
  - proxy-path streaming guarantees
