# Phase 22a.5 Review Fixes

**Date:** 2026-03-21  
**Scope:** Phase 22a.5 interceptor/module streaming review remediation  
**Status:** Implemented and validated in local `dotnet` harnesses

## What was implemented

Addressed the actionable findings from `Development/docs/phases/phase22a/review-22a.5-interceptor-rewrite.md` after the initial 22a.5 completion pass:

1. Hardened cache and observability body-source state for IL2CPP/ARM correctness.
   - `TeeBodySource` now uses `int` state fields plus `Interlocked`/`Volatile` for `_aborted`, `_completedNaturally`, and `_trailersLoaded`
   - `ObservedResponseBodySource` now uses idempotent `Abort()` and `Interlocked.CompareExchange` for `_error`
   - extracted a shared `ResponseBodyDiscardHelper` so retry and redirect now use the same drain/abort/dispose policy

2. Added a test seam for tee write-failure validation without widening public API.
   - introduced internal `IStreamingCacheAccumulator` / `SegmentedBufferStreamingCacheAccumulator`
   - `CacheInterceptor` now owns an internal accumulator factory so tests can inject a failing accumulator
   - added `Runtime/Cache/AssemblyInfo.cs` to expose cache internals to `TurboHTTP.Tests.Runtime`

3. Closed the non-blocking runtime items from the review.
   - redirect discard cancellation now completes as canceled
   - `DecompressionHandler` resolves compression chains per response instead of mutating a handler field
   - buffered decompression now rents its scratch buffer from `ArrayPool<byte>`
   - added explicit sync-over-async and abort-path comments in decompression
   - documented stream-body ownership on redirect replay

4. Closed the missing test gaps.
   - cache tee mid-stream write failure now has a deterministic regression test
   - retry now has a non-buffered streaming drain regression and the two compat-path tests now exercise the real `OnResponseStartAsync` flow
   - metrics now has a full-consumption streaming-byte regression in addition to the partial-consumption case
   - cache streaming tests now use `AssertAsync.Run(...)`

5. Updated the review record itself.
   - `Development/docs/phases/phase22a/review-22a.5-interceptor-rewrite.md` now includes a remediation verification round marking 22a.5 ready for 22a.6

## Decisions / trade-offs

1. **The cache write-failure seam stays internal-only.**  
   The review required a deterministic tee write-failure regression, but this is test infrastructure, not supported public behavior. The accumulator factory and interface remain internal to the cache assembly.

2. **Retry and redirect now share discard behavior through Core, not copy-pasted module code.**  
   Both modules already depend on `TurboHTTP.Core`, so moving the discard logic into `ResponseBodyDiscardHelper` reduced drift without violating asmdef boundaries.

3. **Observation-only review notes were not force-fixed as behavioral changes.**  
   Items like drain-through-decompression, monitor byte-size saturation, and discard-only plugin drain visibility remain accepted informational notes because they were not correctness blockers for 22a.5.

4. **The redirect test warning bucket was not expanded in this pass.**  
   The local review harness still surfaces pre-existing `CS1998` warnings in legacy redirect tests. They did not block behavior validation and were outside the review findings being remediated.
