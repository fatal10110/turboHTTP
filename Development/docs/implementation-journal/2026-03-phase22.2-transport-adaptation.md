# Phase 22.2: Transport Adaptation

**Date:** 2026-03-09  
**Phase:** 22.2 (transport adaptation)  
**Status:** Handler-driven transport path implemented in the transport/core slice; Unity compile/device validation still pending

## What Was Implemented

Phase 22.2 removed the transitional buffered-response bridge from the transport hot path and pushed response delivery fully onto `IHttpHandler` callbacks.

The main runtime changes were:

1. `RawSocketTransport.DispatchAsync` now drives `OnResponseStart` / `OnResponseData` / `OnResponseEnd` directly for HTTP/1.1 and forwards HTTP/2 requests to the new handler-driven `Http2Connection.DispatchAsync`.
2. `Http2Connection` and `Http2Stream` were rewritten so stream completion is handler-driven rather than `UHttpResponse`-backed. Response headers/data/trailers now flow through stream callbacks instead of pooled response-task sources.
3. `ResponseCollectorHandler` was simplified into a pure buffered collector with no `RequestContext` cancellation/error side channel.
4. Request body ownership was removed from the transport layer. `request.DisposeBodyOwner()` no longer appears in `RawSocketTransport` or `Http2Connection`.
5. Testing transports (`MockTransport`, `RecordReplayTransport`, local test doubles) were adapted to emit callbacks directly instead of using the deleted `DeliverResponse` helper.

## Decisions / Trade-Offs

1. **Cancellation contract follows the Phase 22 overview, not the 22.1 bridge behavior:** cancellation now propagates as `OperationCanceledException` with no `OnResponseError` callback.
2. **Synthetic `UHttpResponse.Error` is no longer a buffered-response compatibility path in dispatch mode:** test transports convert it to `OnResponseError(new UHttpException(...))`. This is a deliberate clean break.
3. **Buffered response collection remains available only as an outer helper:** `DispatchBridge`/`TransportDispatchHelper` still exist so `UHttpClient.SendAsync` and selected tests can collect responses, but transports themselves no longer re-emit a prebuilt `UHttpResponse`.
4. **HTTP/2 trailers are not merged into buffered response headers anymore:** the handler contract ends with `OnResponseEnd(trailers, ...)`; `ResponseCollectorHandler` intentionally ignores trailers in the buffered `UHttpResponse`.
5. **Record/replay record mode still buffers the inner response intentionally:** that mode needs the full response body to persist recordings. Passthrough mode now avoids the extra collect-and-replay hop.

### Findings Addressed

| Rubric | Finding | Resolution |
|--------|---------|------------|
| Infrastructure | Transport-owned request body disposal could break retry/redirect redispatch with owner-backed bodies | Removed transport-level `request.DisposeBodyOwner()` from `RawSocketTransport` and `Http2Connection`. |
| Infrastructure | Bridge-era `RequestContext` response/cancellation state increased coupling and hid completion semantics | Removed the stored state and made collector completion depend only on handler callbacks or the dispatch task result. |
| Network | HTTP/2 buffered response assembly on the hot path added extra allocation/copying | Replaced pooled response-task sources with handler-driven `Http2Stream` completion and direct callback delivery. |
| Network | Cancellation/error contract was inconsistent between transport and collector paths | Standardized on: cancellation throws `OperationCanceledException`; non-cancellation transport failures go through `OnResponseError`. |
