# Phase 22.1: Core Interceptor Interfaces

**Date:** 2026-03-08  
**Phase:** 22.1 (Core interceptor interfaces)  
**Status:** In progress (core slice implemented; downstream runtime modules and broad test migration still pending)

## What Was Implemented

Phase 22.1 was started as a clean-break replacement of the middleware pipeline inside `TurboHTTP.Core`. The core client now builds an interceptor-based dispatch chain, transport execution is expressed through handler callbacks, and core-owned adaptive/background behaviors were rewritten to the new interceptor surface.

This pass intentionally does **not** preserve backward compatibility, including test-only compatibility. The deleted core middleware abstractions were not replaced with adapters or shims.

## Decisions Made

1. **Clean-break core API:** `IHttpMiddleware`, `HttpPipeline`, `UHttpClientOptions.Middlewares`, and `PluginContext.RegisterMiddleware` were removed rather than bridged. This applies even to tests.
2. **Buffered bridge is temporary:** transports now expose `DispatchAsync`, but current transport/test implementations still bridge through buffered `UHttpResponse` production where necessary. Native streaming semantics remain phase 22.2 work.
3. **Core must be middleware-free immediately:** core-owned adaptive/background behaviors were moved during 22.1 so `Runtime/Core` no longer references the deleted middleware types.
4. **Queued background requests are cloned:** replay storage now keeps detached request clones to avoid pooled request lifetime hazards.
5. **Synchronous cancellation stays cancellation:** `ResponseCollectorHandler` now preserves `OperationCanceledException` instead of wrapping it as `UHttpException(Unknown)`.

### Findings Addressed

| Rubric | Finding | Resolution |
|--------|---------|------------|
| Infrastructure | Core assembly still contained middleware-era concepts | Removed all middleware/pipeline symbols from `Runtime/Core` and rewrote core-owned adaptive/background pieces. |
| Infrastructure | Background replay queue could retain mutable/pooled request instances | `BackgroundNetworkingInterceptor` stores `request.Clone()` instead of the live request. |
| Network | New dispatch abstraction must preserve cancellation semantics | Transport bridge maps cancellation correctly and the response collector now preserves synchronous cancellation. |

### Deferred Findings

| Rubric | Finding | Target |
|--------|---------|--------|
| Infrastructure | `CapabilityEnforcedInterceptor` does not yet perform full in-flight response mutation comparison, only short-circuit/re-dispatch/out-of-band injection checks | Phase 22 follow-up |
| Infrastructure | `HttpHeaders.Empty` is a mutable shared instance and should be treated carefully in future handler/streaming work | Phase 22 follow-up |
| Network | `RawSocketTransport.DispatchAsync` is still backed by the current buffered send path, so true streaming handler delivery is incomplete | Phase 22.2 |
