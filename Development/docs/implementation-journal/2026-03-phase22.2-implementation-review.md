### New Findings (Round 3, non-blocking)

#### N-1: No regression test for server `RST_STREAM(CANCEL)` → `UHttpException(NetworkError)`

**Source:** Infrastructure architect
**Severity:** P3 (recommended)
**File:** `Tests/Runtime/Transport/Http2/Http2ConnectionTests.cs`

The F-8 fix correctly maps all server RST_STREAM codes to `UHttpException(NetworkError)`, but no test constructs a server-sent `RST_STREAM` frame with error code `CANCEL` (0x8) and asserts the exception type is `UHttpException` with `UHttpErrorType.NetworkError` (not `OperationCanceledException`).

**Recommendation:** Add a targeted test before Phase 22.3 begins.

---

#### N-2: `ReleaseBodyBuffers` relies on `MemoryMarshal.TryGetArray` assumption

**Source:** Infrastructure architect
**Severity:** P3 (informational)
**File:** `Runtime/Transport/Http1/Http11ResponseParser.cs:62-75`

`MemoryMarshal.TryGetArray(Body, out ...)` returns `true` because `ReadFixedBodyAsync` always creates `ReadOnlyMemory<byte>` from a straight `byte[]` rent. If the body allocation path were changed in a future phase (e.g., to use `ArrayPoolMemoryOwner` or `MemoryManager<T>`), `TryGetArray` could return `false` and silently leak.

**Recommendation:** Add a comment documenting the assumption.

## Pass Areas (both reviewers agree, all 3 rounds)

- **Architecture & module dependencies:** No violations. `NullHandler` correctly in Transport, `HandlerCallbackException` correctly `internal sealed` in Core, `TransportDispatchHelper` correctly `public static` in Core. No optional module cross-references.
- **Cancellation contract:** Consistent `OperationCanceledException` / `OnResponseError` split across both transports. Derived cancellation types (e.g., `BackgroundRequestQueuedException`) preserved through `ResponseCollectorHandler.Fail()`.
- **Handler callback ordering:** Correct across all transports: `OnRequestStart` → `OnResponseStart` → `OnResponseData*` → `OnResponseEnd`. Error path: `OnResponseError` (mutually exclusive with `OnResponseEnd`). Cancellation: `OperationCanceledException` with no error callback.
- **Handler fault isolation:** HTTP/2 faults isolated per-stream; `FailFromHandler` + `HasHandlerFault` prevent connection teardown. Handler faults rethrown to caller via `ExceptionDispatchInfo`, not re-reported through `OnResponseError`.
- **IL2CPP/AOT compatibility:** All patterns safe (`Volatile.Read/Write`, `Interlocked`, `ManualResetValueTaskSourceCore<VoidResult>`, static lambdas, `ArrayPool<ResponseEventSignature>`, `ExceptionDispatchInfo`).
- **Buffer lifecycle:** `ParsedResponse` body buffers released in `finally` before pool return. `SegmentedBuffer` in `ResponseCollectorHandler` disposed on error/cancel, detached on success. No leaks on exception paths.
- **Thread safety:** `ResponseCollectorHandler._bodyGate` lock correct. `Http2Stream._responseCompleted` Interlocked exchange + defensive `TrySetResult`/`TrySetException` catch. `_handlerFaulted` via `Interlocked.Exchange`/`Volatile.Read`. ARM IL2CPP memory ordering correct.
- **Http2Stream pool lifecycle:** Ownership transfer correct; `PrepareForPool()`/`Dispose()` non-reentrancy handled via `Interlocked.CompareExchange`. `CancellationRegistration` disposal idempotent.
- **Protocol correctness:** HTTP/1.1 callback ordering matches parse output. HTTP/2 flow control, WINDOW_UPDATE, GOAWAY, RST_STREAM, CONTINUATION handling all correct per RFC 9113. Server RST_STREAM(CANCEL) correctly mapped to `UHttpException(NetworkError)`.
- **Old bridge state keys:** Fully purged (`DeliverResponse`, `SetCancellationException`, `GetCancellationException`, `ResponseErrorStateKey`, `CancellationExceptionStateKey`).
- **Transport body ownership:** `request.DisposeBodyOwner()` fully removed from transport layer.
- **MockTransport/RecordReplayTransport:** Handler emission + response disposal ordering correct per `IHttpHandler` span contract. `RecordReplayTransport.Dispose()` no longer throws.
